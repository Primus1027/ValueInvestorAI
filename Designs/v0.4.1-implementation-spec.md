## 模块 1: `_schemas/seed_v1_1.schema.json`

JSON Schema draft-07，严格校验每条 seed。

关键约束：
- `qualitative_claim` 字段 pattern 禁止含数字（`\\d`）和百分号（`%`）
- `qualitative_claim` 长度 ≤ 300 字符
- `qualitative_claim` 禁止含复合连接词 pattern：`(既.*又|不.*但|同意.*但反对|在.*同时还要求)`
- `quantitative_rule` 和 `qualitative_rule` 至少一个非 null
- `_master` 枚举 `["buffett", "munger", "duan"]`
- re-introduction 字段当 `_reintroduced_from` 存在时，其他 3 个 `_reintroduced_*` 字段全部必填

```python
    Returns {
        "buffett": {"seeds": [...], "metadata": {...}},
        "munger": {...},
        "duan": {...},
        "partial_failure": bool  # True if >= 2 masters fallback
    }
    """
    ...
```

**关键实现细节**：
- 并发调用通过 `concurrent.futures.ThreadPoolExecutor` + subprocess (CliClaudeProvider 模式)
- 每个 subprocess 调 `claude -p --model=claude-sonnet-4-6` + JSON schema 输出
- Retry 时注入更严格的 prompt section（强调"禁止数字、禁止复合"）
- Fallback 时从 `Principles/current.md` 读取该 master 的 HARD 条款
