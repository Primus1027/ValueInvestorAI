## 一、Problem Statement

构建一个 AI 价值投资系统，目标是在 10 年内跑赢指数。系统产出投资决策（具体买入价格和仓位比例）和内容（月度股东信 + 分析文章）。

**核心论点：** AI 在价值投资中的优势不是计算能力，而是情绪中性 + 研究耐力。AI 不会在市场崩盘时恐慌，也不会在读第 200 份年报时感到无聊。

**10x 愿景：** 一个完全由 AI 运营的 Berkshire Hathaway。

**关键洞察（来自创始人）：**
- 大多数"价值投资者"失败不是因为交易时的恐惧/贪婪，而是因为**研究阶段的畏难和无聊**，导致在没有真正看懂公司之前就做出决策
- AI 的另一个优势：它不会因为分析大量资料而感到厌倦，能真正完成人类放弃的深度研究
- 投资大师的灵魂封装 IS the process of discovering investment criteria，标准从深度学习中涌现，不是预先定义的
- "保持谦逊 — 在没有做完所有功课之前，谁也别说自己了解价值投资的全部决策标准"

|------|------|----------|---------|------|
| **巴菲特** | 主投资人 | 所有者收益、护城河分析（品牌/成本/网络/转换）、管理层质量、安全边际、能力圈 | 致股东信 1957-2025 (Berkshire PDF), CNBC Archive | Claude / GPT |
| **芒格** | 对抗性检查 | 多学科思维模型、逆向思维、心理学偏误、检查清单 | 千页合集 PDF, 公开演讲稿 | Claude / GPT |
| **段永平** | 中国视角 | 商业模式简洁性、"做对的事情"、消费品牌评估 | 段永平投资问答录 (雪球特别版 PDF) | Doubao（中文语料权重更高）/ Claude |

**模型选择原则：** 按语料语言分配权重。英文语料为主的大师用 Claude/GPT；中文语料为主的用 Doubao。具体权重在 Phase 1 校准中调优。

**大师的双重角色：**
1. **分析器：** 深度分析通过筛选的公司
2. **筛选器：** 每个大师先做快速判断（"在我的能力圈内吗？值得深入分析吗？"），2/3 大师同意才进入深度分析。标准从大师辩论中涌现，不是单独的规则引擎。

## 三、技术架构

### 技术栈
- **语言：** Python（金融数据生态原生：OpenBB, yfinance, edgartools, FinanceToolkit）
- **数据库：** SQLite + JSON 文件（Phase 0-2）；Phase 2+ 当累积分析 >50 个时加 vector store
- **LLM：** Claude API + GPT API + Doubao API
- **结构化输出：** 使用 Claude/GPT 的 tool_use / function_calling，模型层面保证 JSON schema 合规
- **数据层：** OpenBB (统一抽象), yfinance, SEC EDGAR, Finnhub

### 成本模型

价值投资不是暴力扫描所有公司。真实过程是漏斗：

1. **大师快速筛选（近零成本）：** 3 位大师做快速判断 "在我能力圈内吗？"（~$0.01/公司/大师）。2/3 同意才进入深度分析。
2. **深度分析（LLM 成本）：** 每月仅 2-5 家新公司需要完整的多大师分析。分析一次后，仅在重大事件时更新。
3. **持续扫描（低成本）：** 定期轻量扫描新进入投资范围的公司（价格下跌、业绩改善、新 IPO）。

**月预算：** ~$50-100/月 LLM APIs。

## 四、分阶段实施计划

### Phase 0: 基础搭建（Week 1）

| 任务 | 内容 | 可并行 |
|------|------|--------|
| Memory 层 | SQLite schema + JSON 格式 + 读写接口 | 是 |
| Data 层 | OpenBB/yfinance 接入 + 基本财务数据获取 | 是 |
| 资料收集 | 下载巴菲特股东信、芒格合集 PDF、段永平问答录 | 是 |
| 项目初始化 | Python 项目结构、依赖安装、git repo | 是 |
| 校准案例 | 确认 5 个历史案例 + 2-3 个对抗性案例 | 是 |

**Exit Criteria:**
- [ ] Memory 接口可读写，Buffett 分析可以跨 session 存储和检索（包含：公司名、估值范围、关键护城河因素、分析日期；检索 <10 秒）
- [ ] Tech stack 搭建完成，repo 初始化
- [ ] 5 个校准案例 + 2-3 个对抗性案例已确认并有文档化的真实大师立场
- [ ] 3 位大师的源资料已下载整理

### Phase 1: 决策风格建模（Week 2-4）

3 位大师的决策框架并行构建。

**灵魂封装分两步：**
1. **Phase 1a（快速）：** AI 从现有训练数据生成第一版框架（AI 已经"读过"这些资料），产出初始决策标准
2. **Phase 1b（深化）：** 喂入真实源资料（PDF/转录稿），深化和纠正框架，发现 AI 训练数据中缺失的细微标准

**校准协议：**
- 5 个常规历史案例：大师的方向性判断（买/卖/不碰）
- 2-3 个**对抗性案例**：大师的哲学与实际行为矛盾的案例（如巴菲特买航空股，违反他数十年的反航空立场）。测试推理能力，不是记忆力
- **危机场景测试：** 喂入 2020 年 3 月的市场数据 + 新闻标题，验证大师是否保持纪律（"在别人恐惧时贪婪"）还是被新闻情绪带偏。测试核心论点

**结构化输出：** 每个大师的分析通过 tool_use 输出结构化 JSON：
```json
{
  "ticker": "AAPL",
  "master": "buffett",
  "recommendation": "buy",
  "fair_value_range": [150, 180],
  "pe_ratio": 28.5,
  "moat_type": "brand + ecosystem",
  "moat_durability": "high",
  "key_risks": ["regulatory pressure", "China dependency"],
  "position_size_suggestion": "5-8% of portfolio",
  "confidence": "high",
  "reasoning_summary": "..."
}
```

**Exit Criteria:**
- [ ] 3 位大师分析同一家公司产出可测量的不同视角
- [ ] 每位大师识别 ≥3 个框架特有的因素（不是通用清单）
- [ ] 每位大师提出估值范围并明确陈述假设
- [ ] 校准测试：常规案例 ≥4/5 方向准确
- [ ] 对抗性案例：能解释为什么大师在这些案例中偏离了自己的哲学
- [ ] 危机场景：大师在恐慌环境下保持价值投资纪律
- [ ] Memory 系统跨 session 保留并检索先前分析

### Phase 2: 分析引擎（Week 5-8）

Phase 1 大师通过校准后启动。

- **大师筛选：** 候选公司先过 3 位大师的快速判断，2/3 同意进入深度分析
- **深度分析：** 每月 2-5 家公司，全部大师分析 + 数据验证
- **跨大师综合：** 产生组合视角的原创投资论点
- **内容生成：** 每次分析产出 1 篇文章（1000-2000 字），结构：论点 + 证据 + 大师视角 + 分歧亮点
- **月度股东信：** 旗舰内容格式，总结 30 天的 AI 董事会思考（巴菲特的格式）
- **反投资组合：** 记录研究过但拒绝的公司 + 拒绝原因 + 持续追踪市场数据验证
- **论点演变时间线：** 每家分析过的公司，维护时间戳化的论点历史
- **审计追踪：** 结构化 JSON 记录每次分析

**Exit Criteria:**
- [ ] 分析流水线连续运行 3 周
- [ ] ≥10 家公司分析完成，有具体推荐
- [ ] 内容生成产出符合要求的文章（论点+证据+大师视角+分歧），1000-2000 字，创始人审核 ≤2 轮修改通过
- [ ] 审计追踪以可查询格式捕获每次分析
- [ ] 产出第一封月度股东信

### Phase 3: 投资决策（Week 8+）

**前置条件：** 创始人个人法律舒适度。合规意见建议但不阻塞个人资金投资。

- 具体买入价格 + 组合配置比例
- 人工审批门（创始人审核每笔买卖 — 审核推理 IS 验证方法）
- 持仓追踪 + 论点监控（每周人工审查）
- **自我改进反馈循环：** 每个投资决策追踪结果 → 复盘分析（不只是"股价跌了"，而是"我识别的护城河比我想的弱，因为竞争对手 X 推出了 Y，下次我应该关注什么"） → 跨复盘的模式识别 → 回馈 Memory 层

## 五、前提假设

1. AI 可以学习并持续应用价值投资原则
2. 人类在价值投资中失败是因为情绪 + 研究疲劳 — AI 消除两者
3. 内容生产和投资决策有协同效应 — 写文章就是阐述投资逻辑
4. 创始人跳过虚拟组合，直接用真实资金（$100K），每笔交易需人工审核
5. Memory 架构是关键技术挑战 — AI 必须学会什么该记、什么该忘
6. 多大师的价值不在共识，在于**分歧检测** — 分歧强制更深入分析
7. 投资过程从第一天就是产品 — 即使早期分析不够锐利，可见的自我进化过程本身就有价值
8. **AI 默认会"污染"为股票研究模式**（conviction scoring、季度更新）而非价值投资模式 — 灵魂封装的核心目的之一就是防止这种模式污染

## 六、不做列表

- SEC 投资顾问注册（推迟到获得法律意见）
- LLM 微调（仅在 RAG/prompt 不够时探索）
- 券商 API 集成（所有交易手动执行）
- 多平台发布基础设施（Phase 1-2 仅个人使用）
- 回测引擎（不需要 — 前瞻性决策）
- 移动应用 / Web 仪表盘（先 CLI）
- 做空 / 期权策略（仅做多）
- 社区输入决策（社区仅围观）
- 李录独立大师（资料不足以校准，已删除）
- Conviction scoring 系统（股票研究模式，不是价值投资模式）
- 向量数据库（Phase 2+ 当 >50 个分析时再加）
- 独立规则引擎筛选（大师即筛选器）

## 七、成功标准

- [ ] 3 位大师通过校准测试（常规 ≥4/5 + 对抗性案例 + 危机场景）
- [ ] ≥10 家公司分析，有具体买/持/卖 + 价格目标 + 配置比例
- [ ] 分析引擎自主运行 4+ 周
- [ ] 创始人审核推理后愿意用真实资金投资
- [ ] Memory 系统展示学习能力：后续分析同一公司时包含从早期分析中累积的知识
- [ ] 至少一个案例中大师分歧导致了比任何单个大师更好的决策
- [ ] 反投资组合中至少追踪 5 家被拒绝的公司的后续表现
- [ ] 产出第一封月度股东信

## 八、失败模式

| 失败 | 影响 | 缓解措施 |
|------|------|---------|
| 数据 API 不可用 | 分析流水线停滞 | 回退到缓存数据（24h 陈旧对价值投资可接受） |
| LLM 幻觉财务数据 | 错误分析，可能导致错误投资 | 结构化输出 + 自动对比源 API 数据，标记不匹配 |
| LLM 拒绝回答 | 分析不完整 | 检测拒绝 → 重试或标记为需要人工处理 **← 关键缺口** |
| 内容低于质量阈值 | 不可发布的文章 | 跳过发布，记录待审查；创始人审核门兜底 |
| Memory 系统丢失上下文 | 大师重新开始，丢失累积学习 | SQLite 每日备份 + 定期完整性检查 |
| SQLite 锁定 | 并发写入失败 | 序列化写入操作 **← 关键缺口** |
| 表演性一致（performative agreement） | 综合引擎输出看似合理但无独立视角 | 强制独立信息子集 + 检测 **← 关键缺口** |
| 成本超支 | 月预算超支 | 硬消费上限 + 警报；降级为仅周度分析 |

## 九、并行开发策略

```
Lane A (并行): Memory 层 + Data 层        ← 无依赖，立即启动
Lane B (并行): 大师源资料收集              ← 无依赖，立即启动
Lane C (串行): 大师 Agents → 验证 → 综合 → 内容  ← 依赖 Lane A
Lane D (并行设计/串行执行): 校准测试设计(并行) → 执行(依赖 Lane C)
```
